# Alliance Prototype Project – E-Commerce Platform Prototype (Simulation Task)

## 1. Project Overview

The Alliance Prototype Project is an **11-week simulation task** completed as part of a structured project management curriculum at The Catholic University of America. The objective was to plan, manage, and deliver a functional e-commerce prototype for a fictional client, **Uniworld**, using real-world project management methodologies.

This repository documents the **entire project lifecycle** planning, execution, monitoring, and closure supported by real simulation outputs, charts, and cost reports.



## 2. Objectives

### Business Objectives (Simulation)
- Modernize Uniworld’s sales and marketing capabilities through an e-commerce prototype.
- Demonstrate feasibility for a full production rollout.
- Provide stakeholders with clear visibility into scope, schedule, and budget.

### Project Objectives
- Deliver a functional prototype within **11 weeks**.
- Stay within a **$50,000** simulated budget.
- Maintain on-time delivery with minimal known defects at acceptance.



## 3. Scope

### In Scope
- Requirements gathering and stakeholder alignment (simulated)
- Prototype design (catalog, cart, checkout flow)
- WBS, schedule, and resource planning
- Risk management and mitigation
- Quality management and defect tracking
- Final stakeholder presentation



## 4. Budget & Cost Performance

The project was delivered **under budget** and **on schedule**.

- **Actual progress achieved in terms of project deliverables**

**Schedule Performance Index (SPI)**: Indicates the efficiency of the project schedule
- **SPI = EV / PV = 1**

**Cost Performance Index (CPI)**: Measures the cost efficiency of the project.

- **CPI = EV / Actual Cost = 1.039**
